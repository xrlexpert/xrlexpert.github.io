    2. γ=0\gamma =0γ=0：智能体只关心当前奖励，忽略未来奖励。

  1. Episode(or a trial): 智能体（Agent）在环境中按照某个策略（Policy）进行交互，直到到达某个终止状态（Terminal State）就停止的完整轨迹。
    • 由终止状态的任务称为 episodic tasks
    • 无终止状态的任务称为 continuing tasks

Markov decision process （MDP）

1. 集合（Sets）

  • 状态（State）：状态集合 SSS。
  • 动作（Action）：在状态 s∈Ss \in Ss∈S 下，可用的动作集合 A(s)A(s)A(s)。
  • 奖励（Reward）：在状态 sss 下采取动作 aaa 后，可能获得的奖励集合 R(s,a)R(s, a)R(s,a)。

2. 概率分布（Probability Distributions）

  • 状态转移概率（State Transition Probability）：

    • 在状态 sss 下采取动作 aaa，转移到状态 s′s's′ 的概率为：

      p(s′∣s,a)p(s' \mid s, a) p(s′∣s,a)

  • 奖励概率（Reward Probability）：

    • 在状态 sss 下采取动作 aaa，获得奖励 rrr 的概率为：

      p(r∣s,a)p(r \mid s, a) p(r∣s,a)

3. 策略（Policy）

  • 在状态 sss 下，选择动作 aaa 的概率为：

    π(a∣s)\pi(a \mid s) π(a∣s)

4. 马尔可夫性质（Markov Property）

  • 无记忆性（Memoryless Property）：

    • 下一个状态 st+1s_{t+1}st+1​ 和奖励 rt+1r_{t+1}rt+1​ 只依赖于当前状态 sts_tst​ 和动作 ata_tat​，而与之前的状态和动作无关。

    • 数学表示为：

      p(st+1∣st,at,st−1,at−1,…,s0,a0)=p(st+1∣st,at)p(s_{t+1} \mid s_t, a_t, s_{t-1}, a_{t-1}, \dots, s_0, a_0) = p(s_{t+1} \mid s_t, a_t) p(st+1​∣st​,at​,st−1​,at−1​,…,s0​,a0​)=p(st+1​∣st​,at​)

      p(rt+1∣st,at,st−1,at−1,…,s0,a0)=p(rt+1∣st,at)p(r_{t+1} \mid s_t, a_t, s_{t-1}, a_{t-1}, \dots, s_0, a_0) = p(r_{t+1} \mid s_t, a_t) p(rt+1​∣st​,at​,st−1​,at−1​,…,s0​,a0​)=p(rt+1​∣st​,at​)

5. MDP 框架

  • 所有上述概念都可以放在 MDP 框架中：
    • 状态、动作、奖励 是 MDP 的基本组成部分。
    • 状态转移概率 和 奖励概率 定义了环境的动态特性。
    • 策略 是智能体的行为规则。
    • 马尔可夫性质 是 MDP 的核心假设，简化了问题的复杂性。
